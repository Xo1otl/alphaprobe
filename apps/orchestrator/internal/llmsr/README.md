# 導入

従来の記号的回帰（Symbolic Regression）では、構文木を用いた探索が主流であった。しかし、このアプローチは探索空間が限定的であり、人間が事前に定義した演算子の組み合わせを超える、全く新しいプログラム構造を発見することが困難であるという課題があった。

# 概要

`llmsr`は、この課題を解決するため、大規模言語モデル(LLM)の広範な知識を活用してプログラムを発見するモジュールである。そのアーキテクチャは、**プログラムスケルトン生成**と**パラメータ最適化**という、責務が明確に分離された2つの主要コンポーネントで構成される。これにより、LLMが得意とする大局的な構造探索と、数値最適化アルゴリズムが得意とする局所的なパラメータチューニングを組み合わせ、効率的なプログラム探索を実現するものである。

# プログラムスケルトン生成

**プログラムスケルトン生成**は、`llmsr`の第一段階を担うコンポーネントである。大規模言語モデル(LLM)に対し、問題定義や既存の有望なコードサンプルを含むプロンプトを提示し、改善された新しいプログラムの骨格（スケルトン）を提案させる。このスケルトンは、具体的な係数を含まないテンプレート形式の関数であり、探索空間における新たな構造的候補となる。

# パラメータ最適化

**パラメータ最適化**は、第二段階を担うコンポーネントである。`プログラムスケルトン生成`によって提案されたスケルトンを受け取り、その中に含まれる未定の係数（パラメータ）を数値オプティマイザ（例: BFGS, Adam）で決定する。オプティマイザは、与えられたデータセットに対する誤差が最小になるようパラメータを調整する。最適化が完了したプログラムは評価され、その性能がスコアとして確定する。
