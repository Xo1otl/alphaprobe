# orchestrator

## 概要

`orchestrator`は、フレームワークの中核を担うコマンドサービスです。
汎用的な並列実行基盤 (`pipeline`) と、それを利用したアルゴリズムフレームワーク (`islandga`) を提供し、I/Oバウンドなタスクのオーケストレーションを安全かつクリーンに記述することを目的とします。

## 設計原則

本フレームワークは、コードの安全性、予測可能性、テスト容易性を最大化するため、以下の原則に基づき設計されています。

-   **状態の一元管理:** 探索や実行に関する全ての状態は、単一の`State`構造体に集約されます。これにより、状態の参照や更新が予測可能になります。また、`State`オブジェクトをシリアライズ/デシリアライズすることで、中断・再開といった機能の実装が容易になります。

-   **副作用のない純粋関数:** 利用者が注入するビジネスロジック (`Propose`, `Observe`等) は、副作用のない純粋関数であることを前提とします。入力データを変更せず、常に新しいデータを生成して返すことで、システム全体の状態を健全に保ちます。

-   **I/Oバウンド処理の重視:** 各処理ステップは、API連携やシミュレーション実行といった完了までに時間を要するI/Oバウンドな処理を想定しています。そのため、データコピー等のわずかな計算コストよりも、アーキテクチャ全体の堅牢性とコードの明確性を優先します。

## 主要コンポーネントと利用契約

### `pipeline`
ジェネリクスを用いた、汎用的な並列パイプライン実行基盤です。ワーカープールと制御ループの機能を提供します。

### `islandga`
`pipeline`を基盤として構築された、汎用的な島モデル遺伝的アルゴリズムのフレームワークです。
`islandga`を利用する際は、以下の契約を遵守する関数を外部から注入 (DI) します。

-   `ProposeFunc`: **入力データを変更せず**、必ず**新しい独立した遺伝子 (`Gene`) を生成**して返します (MUST)。
-   `ObserveFunc`: 与えられた遺伝子を評価し、結果を返します。副作用を持ってはなりません (MUST NOT)。
-   `CloneFunc`: フレームワークが移住処理などで遺伝子の安全なコピーを必要とする際に使用されます。これにより、コピーの責務が明確化されます。
